1.What is git?

* Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency.

* Version control or source control, is the practice of tracking and managing changes to software code.This Version control systems are software tools that help software teams manage changes to source code over time.

* Types of Version Control systems include:

  1. Local Version Control Systems- Changes are tracked locally on a developer's machine
  2. Centralized Version Control Systems -Developers check out the latest version from the repository, make changes, and then check in their changes.
  3. Distributed Version Control Systems -Excellent for large teams and projects that require frequent branching and merging.

*This are some of the use-cases of Git:

  • Tracking Changes
  • Collaboration
  • Non-linear Development
  • Backup and Recovery
  • Project organization

6.Adding files to the staging area

*Wondering what staging is -The staging area is a file, generally contained in your Git directory, that stores information about what will go into your next commit

* git add [filename] selects that file, and moves it to the staging area, marking it for inclusion in the next commit. You can select all files, a directory, specific files, or even specific parts of a file for staging and commit.

*what is the difference between tracked and untracked files

Tracked files are the files git knows about while untracked git knows nothing about them

8. Branching

*A branch represents an independent line of development. Branches serve as an abstraction for the edit/stage/commit process.

*You can think of them as a way to request a brand new working directory, staging area, and project history.

9.Creating a branch
